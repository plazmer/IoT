<h1 id="unwired-range">Unwired Range<a class="headerlink" href="#unwired-range" title="Permanent link">&para;</a></h1>
<h2 id="lora">Радиочастотные сети LoRa<a class="headerlink" href="#lora" title="Permanent link">&para;</a></h2>
<p>Unwired Range — это семейство устройств с поддержкой технологии LoRa, радиочастотной модуляции, позволяющей устанавливать двустороннюю передачу данных на расстоянии до 30 км, не выходя за ограничения так называемых безлицензионных (не требующих получения разрешений на эксплуатацию радиопередающих устройств) радиодиапазонов. На данный момент в России основными диапазонами для сетей LoRa являются:</p>
<ul>
<li>864,0 — 865,0 МГц с эффективной мощностью излучения до 25 мВт, с рабочим циклом не более 0,1 % (длительность передачи 3,6 с/час) или использованием режима «прослушивание перед передачей»</li>
<li>868,7 — 869,2 МГц с эффективной мощностью излучения до 25 мВт, без ограничения на рабочий цикл.</li>
</ul>
<p>Хотя теоретически LoRa может использовать весь спектр 137 — 1020 МГц, на практике прочие диапазоны не представляют интереса:</p>
<ul>
<li>низкочастотные диапазоны требуют больших размеров приёмопередающих антенн</li>
<li>в диапазоне 443 МГц в России мощность передатчика ограничена 10 мВт</li>
<li>диапазон 446 МГц с мощностью до 0,5 Вт предназначен только для портативных радиостанций</li>
<li>...и так далее.</li>
</ul>
<p>Что касается диапазона 864,0 — 865,0 МГц, то в российском законодательстве на данный момент не расшифровывается понятие «прослушивание перед передачей» (LBT, listen before talk). Тем не менее, в соответствующем постановлении ГКРЧ оно упоминается, так что при необходимости использовать этот режим разумно будет ориентироваться на положения, установленные европейскими стандартами (EN 300 220-1), так как в целом в рамках гармонизации использования РЧ-спектра неясности в российских НПА, как правило, разрешаются в пользу большей совместимости с НПА европейскими:</p>
<ul>
<li>порог определения занятости радиоэфира — -90 дБм</li>
<li>задержка между прослушиванием эфира и началом передачи — не более 5 мс</li>
<li>задержка между двумя последовательными актами передачи — не менее 100 мс</li>
<li>длительность единичной передачи — не более 1 с</li>
<li>длительность двустороннего радиообмена — не более 4 с</li>
<li>накопленное время радиопередачи — не более 100 с/час</li>
</ul>
<p>Важным является тот факт, что указанные ограничения распространяются как на конечные устройства (радиомодемы), так и на базовые станции (граничные роутеры). Последние также должны соответствовать требованиям по использованию РЧ-спектра, что может стать особенно важным в радиосетях с двусторонним радиообменом при большом числе конечных устройств, приходящихся на одну базовую станцию.</p>
<p>В прошивках Unwired Range при выборе региона «Russia» используются частотные каналы 868,85 МГц и 869,05 МГц.</p>
<h2 id="_1">Топология сети<a class="headerlink" href="#_1" title="Permanent link">&para;</a></h2>
<p>Сама по себе технология LoRa определяет лишь физический уровень канала связи. Традиционно внедряются два типа сетей на базе технологии LoRa:</p>
<ul>
<li>сети топологии «звезда» с единственной базовой станцией, использующей MAC-уровень собственной разработки;</li>
<li>сотовые сети («звезда звёзд») с множественными базовыми станциями, использующими MAC-уровень LoRaWAN и единый сетевой сервер, обеспечивающий управление сетью.</li>
</ul>
<p>Сотовые сети, часто называемые просто «LoRaWAN», лучше всего подходят для покрытия крупных объектов, масштаба городских районов, с предоставлением услуг связи сторонним абонентам, обеспечением роуминга устройств между базовыми станциями и т.п.</p>
<p>Для объектов ограниченной площади (масштаба одного квартала, торгового или делового здания, заводского цеха, сельскохозяйственного поля) интересны более простые в установке, настройке и поддержке сети топологии «звезда», не использующие протокол LoRaWAN.</p>
<p>Модули, устройства и базовые станции Unwired Range на данный момент поддерживают топологию «звезда» с MAC-уровнем, разработанным в компании Unwired Devices. Базовая станция UNWD-BASE является конечным элементом такой сети и не требует для функционирования каких-либо внешних сервисов. Радиочастотный протокол поддерживает двустороннюю связь, шифрование AES-256, имитозащиту и индивидуальную адресацию всех устройств, находящихся в сети.</p>
<h2 id="unwired-range_1">Радиомодули Unwired Range<a class="headerlink" href="#unwired-range_1" title="Permanent link">&para;</a></h2>
<p>Базовым модулем связи является модем Unwired Range, поставляемый либо в виде отдельного электронного модуля, либо в составе готового устройства.</p>
<p>Модемы Unwired Range выпускаются в двух форм-факторах, отличающихся установочными размерами и типом разъёма.</p>
<p><img alt="LoRa-модемы Unwired Range" src="images/unwired-range-pcb.png" /></p>
<p>Первый вариант предназначен для горизонтальной установки на печатную плату основного устройства и обладает разъёмом U.fl для подключения антенны, второй — преимущественно для вертикальной установки, в том числе в корпуса для крепления на DIN-рейку, и обладает антенным разъёмом SMA-F или несъёмной штыревой антенной.</p>
<p>И те, и другие модули построены на комбинации из микроконтроллера STM32L151CC и приёмопередатчика SX1276, программно совместимы друг с другом и имеют одинаковый набор внешних сигналов.</p>

<h2 id="_2">Настройка параметров радиомодуля<a class="headerlink" href="#_2" title="Permanent link">&para;</a></h2>
<p>Для первичной настройки радиомодуля используется отладочная консоль на интерфейсе UART (сигналы DIO2, DIO3).</p>
<ol>
<li>
<p>Установите модем LoRa в адаптер UMDK-RFU со встроенным мостом USB-UART</p>
</li>
<li>
<p>Подключите адаптер к компьютеру кабелем microUSB, включите питание адаптера переключателем «Power» дождитесь окончания установки драйвера</p>
</li>
<li>
<p>Запустите программу терминала (например, Termite) и откройте нужный COM-порт, выбрав параметры 115200-8N1, без контроля потока. Терминал должен поддерживать полный перенос строки с возвратом каретки (CR-LF) для устройств, передающих строки только с символом переноса (LF).</p>
</li>
</ol>
<p><img alt="Адаптер UMDK-RFU с модемом LoRa" src="images/SAM_7452.jpg" />
<img alt="Адаптер UMDK-RFU с модемом LoRa" src="images/SAM_7454.jpg" /></p>
<p>При корректном подключении в терминале появится заставка прошивки радиомодуля.</p>
<p>Если радиомодуль ранее не настраивался или его настройки были полностью очищены, прошивка потребует ввести идентификационный номер радиомодуля EUI64. Номер напечатан на этикетке на нижней стороне радиомодуля и представляет собой 64 бита в 16-ричном коде.</p>
<p>Устанавливается номер командой <code>set</code>, сохраняется — командой <code>save</code>:</p>
<pre><code>set eui64 807b859020000001
save
</code></pre>

<p>Если у вас есть собственный диапазон адресов EUI64 — вы можете использовать значения из него. В противном случае настоятельно рекомендуется сохранять адрес EUI64, на который модуль был настроен изначально. В пределах одной сети каждый радиомодуль должен иметь уникальный адрес; использование EUI64 из адресного пространства, официально выделенного организацией IEEE, гарантирует это, так как исключает выпуск двух устройств с одинаковыми адресами EUI64, независимо от их модели и производителя.</p>
<p>После перезагрузки модуль предложить ввести уникальный номер APPID64, ключ шифрования сети и ключ Nonce.</p>
<p>APPID64 — уникальный идентификатор приложения, работающего на радиомодуле. APPID64 может использоваться на шлюзе или сервере сети LoRaWAN для определения, что именно за данные поступают с этого радиомодуля; при этом у множества радиомодулей, выполняющих одну и ту же задачу, может быть один и тот же APPID64.</p>
<p>В случае, если APPID64 не используется, он может быть любым, например:</p>
<pre><code>set appid64 0000000000000001
</code></pre>

<p>JOINKEY — ключ шифрования, используемый в данной сети. JOINKEY должен быть абсолютно одинаковым для всех устройств сети, а также для базовой станции.</p>
<p>Ключ имеет разрядность 128 бит и задаётся в виде шестнадцатиричного числа (всего 32 символа):</p>
<pre><code>set joinkey 918771929bbab19898c8a787f70a8810
</code></pre>

<p>NONCE — ключ шифрования для устройств со статической авторизацией в сети LoRa. При использовании динамической адресации (по умолчанию) он может быть любым.</p>
<pre><code>set nonce abcdabcd
save
</code></pre>

<p>После перезагрузки модуль предложить перейти к настройкам радиочастной части.</p>
<p>Для модуля базовой станции это:</p>
<ul>
<li>region — регион, которые определяются допустимые частотные диапазоны (0 — Европа, 1 — Россия)</li>
<li>ch — частотный канал (в Европе — 868,1, 868,3 или 868,5 МГц, в России — 868,85 или 869,05 МГц)</li>
<li>dr — скорость передачи данных (dr 0 соответствует минимальной скорости и SF=12, dr 7 — максимальной скорости; дальность связи уменьшается с увеличением скорости)</li>
</ul>
<p>Для радиомодуля оконечного устройства добавляются параметры:</p>
<ul>
<li>maxretr — максимальное количество попыток передать данные (по умолчанию модули Unwired Range работают с подтверждением доставки пакетов)</li>
<li>class — тип устройства. Устройства класса A включают приёмник только на короткий промежуток времени после завершения передачи, устройства класса B могут включать приёмник по расписанию, устройства класса C включены всегда.</li>
</ul>
<p>В текущей версии прошивок класс B не поддерживается, при его выборе устройство будет работать аналогично классу C.</p>
<p>Устройства класса А между событиями передачи данных уходят в глубокий сон для экономии энергии, поэтому сменить их параметры из консоли невозможно. Для смены параметров необходимо запустить модуль в безопасном режиме, удерживая кнопку Connect на плате UMDK-RF при нажатии кнопки Reset — тогда модуль запустится в классе C.</p>
<p>Полный список настроек можно посмотреть, дав команду <code>set</code> без параметров, полный список команд — дав команду <code>help</code>.</p>

<p>После подключения модуля, сохранения настроек командой <code>save</code> и перезагрузки в системе команд модуля появятся дополнительные команды, одноимённые включённым драйверам. При вызове такой команды без параметров она сообщит полный возможный список параметров.</p>
<p>Конфигурация радиомодуля может быть сброшена командой <code>clear</code> с параметрами <code>all</code>, <code>key</code> или <code>modules</code>.</p>
<ul>
<li>clear all сбрасывает всю конфигурацию модуля, включая ключи шифрования и EUI64</li>
<li>clear keys сбрасывает только ключ шифрования</li>
<li>clear modules сбрасывает настройки драйверов (как включённых в данный момент, так и выключенных)</li>
</ul>
<p>В случае, если включённый драйвер привёл к неработоспособности прошивки, необходим установить модуль в адаптер UMDK-RF, включить питание, а потом, удерживая кнопку Connect адаптера, коротко нажать кнопку Reset. Модем загрузится в безопасном режиме — без инициализации драйверов. После этого необходимо отключить проблемный драйвер, сохранить настройки и перезагрузить модем в обычном режиме.</p>
<p>Настройки радиочасти и список включённых драйверов можно изменять без сброса настроек модуля. Все настройки применяются после их сохранения и последующей перезагрузки командами save и reboot.</p>
<h2 id="unwired-range_3">Обновление прошивки модуля Unwired Range<a class="headerlink" href="#unwired-range_3" title="Permanent link">&para;</a></h2>
<p>Обновление прошивки можно произвести двумя способами — либо с помощью JTAG/SWD-программатора UMDK-JTAG (или иного с поддержкой процессоров STM32L1), либо с помощью встроенного загрузичка модуля (только для модулей ревизии R170105, она же r3, и адаптеров UMDK-RF версии 1.05 и новее).</p>
<h3 id="_3">Обновление с помощью встроенного загрузчика<a class="headerlink" href="#_3" title="Permanent link">&para;</a></h3>
<ol>
<li>Скачайте программу <a href="https://files.unwds.com/files/range/firmware/stm32flash.exe">stm32flash</a> и установите её</li>
<li>Подключите адаптер UMDK-RF со встроенным мостом USB-UART к компьютеру</li>
<li>Установите на адаптер радиомодем Unwired Range</li>
<li>Включите питание UMDK-RF</li>
<li>Зажмите кнопки Reset и Boot, отпустите сначала Reset, а потом — Boot</li>
<li>Запустите программу stm32flash со следующими параметрами: <code>stm32flash -R -E -w &lt;firmware.hex&gt; COMxx</code>, подставив правильный номер COM-порта и название файла с прошивкой</li>
<li>Нажмите «Next» и убедитесь, что светофор показывает зелёный, а надпись рядом с ним гласит «Target is readable»</li>
<li>Нажмите «Next». В поле «Target» должен сам появиться контроллер «STM32L1_Cat3-256K»</li>
<li>Нажмите «Next». Выберите раздел «Download to device», откройте HEX-файл с прошивкой и нажмите «Next»</li>
<li>Процесс прошивки занимает 15-30 секунд, в зависимости от размера прошивки </li>
</ol>
<p>Внимание! Подходит только версия stm32flash с сайта Unwired Devices, так как оригинальная утилита некорректно работает с контроллерами STM32L151CC. Если вам необходима эта версия для других ОС, её исходные коды доступны на <a href="https://github.com/unwireddevices/stm32flash">GitHub</a>.</p>
<h3 id="umdk-jtag">Обновление с помощью программатора UMDK-JTAG<a class="headerlink" href="#umdk-jtag" title="Permanent link">&para;</a></h3>
<p>UMDK-JTAG — программатор с поддержкой интерфейсов JTAG/SWD и встроенным сервером gdb. Для работы с ним необходимо установить <a href="https://developer.arm.com/open-source/gnu-toolchain/gnu-rm">GNU ARM Toolchain</a>, включающий в себя клиент gdb.</p>
<p>Драйвер UMDK-JTAG для Windows — <a href="https://files.unwds.com/files/range/firmware/serial_install.exe">usb_serial</a>.</p>
<p>После установки драйвера и подключения программатора в системе появятся два виртуальных COM-порта — один из них будет соответствовать встроенному в программатор адаптеру USB-UART, второй — серверу GDB. К сожалению, Windows показывает для обоих устройств одинаковые называния «Teensy USB Serial» в диспетчере устройств для обоих портов, поэтому, чтобы их различить, необходимо открыть свойства одного из устройств, выбрать вкладку «Сведения» и посмотреть пункт «Описание устройства, предоставленное шиной». Там будет значится либо «Black Magic UART Port», либо «Black Magic GDB Server».</p>
<p>Запуск клиента gdb из командной строки (под Windows удобнее всего использовать окружение MinGW):</p>
<pre><code>arm-none-eabi-gdb lora-star-uni.hex
</code></pre>

<p>Подключение клиента gdb к программатору осуществляется следующими командами gdb:</p>
<pre><code>target extended-remote \\.\COM16
mon conn enable
mon swdp_scan
attach 1
</code></pre>

<p>В первой команде в качестве номера COM-порта необходимо указать соответствующий порту Black Magic GDB Server.</p>
<p>Загрузка HEX-файла в микроконтроллер модема и выход из gdb:</p>
<pre><code>load
quit
</code></pre>

<p>Чтобы не вводить команды каждый раз вручную, удобно в рабочей папке создать файл gdbinit со следующим содержанием:</p>
<pre><code>target extended-remote \\.\COM16
mon conn enable
mon swdp_scan
attach 1
load
quit
</code></pre>

<p>И запускать gdb командой:</p>
<pre><code>arm-none-eabi-gdb -x gdbinit lora-star-uni.hex
</code></pre>

<p>Файл прошивки не обязательно должен находиться в той же папке, к нему можно указать более длинный путь.</p>
<p>При использовании адаптера UMDK-RF со встроенным USB-UART необходимо учитывать, что он будет конфликтовать с USB-UART программатора UMDK-JTAG. Если необходимо программирование модема и доступ к его консоли (например, в ходе отладки новой прошивки), надо соответствующие консольному UART ножки UMDK-JTAG (2 и 3) отогнуть на несколько градусов в сторону, чтобы они не попали в разъём UMDK-RF:</p>
<p><img alt="Адаптер UMDK-RF с адаптером UMDK-JTAG" src="images/using_jtag_adapter.jpg" /></p>
<p>При этом консоль модема будет доступна через адаптер, встроенный в UDMK-RF, а адаптер UART из UMDK-JTAG может использоваться для других целей — например, при отладке драйверов, использующих UART на ножках 25-26, вход RX UMDK-JTAG можно подключать к RX или TX этого UART для целей мониторинга.</p>
<h2 id="unwd-base-r1">Работа с базовой станцией UNWD-BASE R1<a class="headerlink" href="#unwd-base-r1" title="Permanent link">&para;</a></h2>
<p><em>NB: ниже описывается работа с базовой станцией UNWD-BASE, однако на практике базовая станция может быть построена на практически любом компьютере под ОС Linux с интерфейсом UART, например, Raspberry Pi, при условии сборки для него пакета <a href="https://github.com/unwireddevices/lora-mqtt">lora-mqtt</a> и наличия на нём MQTT-брокера mosquitto.</em></p>
<p>Базовая станция UNWD-BASE представляет собой комбинацию из радиомодуля Unwired Range и микрокомпьютера с ОС OpenWRT Linux. Задача микрокомьютера — поддерживать работу радиочастотной сети, включая механизмы аутентификации, авторизации и шифрования, а также транслировать компактный бинарный протокол радиообмена в сообщения в формате JSON для установленного на микрокомпьютере MQTT-брокера.</p>
<p>Подключиться к UNWD-BASE можно по сети Ethernet (один из портов по умолчанию настроен как WAN с DHCP-клиентом, второй — LAN с DHCP-сервером) или Wi-Fi — сеть называется UNWD-BASE-NNNNNN, где NNNNNN — последние 6 символов MAC-адреса интерфейса, пароль — unwiredbase</p>
<p>Базовая станция имеет веб-интерфейс для настройки сетевых интерфейсов, в том числе для подключения к существующим сетям Wi-Fi, доступный по адресу 192.168.4.254 с логином root и паролем admin.</p>
<p>Для работы с MQTT и конечными устройствами радиосети необходим доступ к консоли по ssh — адрес 192.168.4.254, порт 22, пароль и логин такие же (NB: при первой загрузке после обновления прошивки базовая станция не будет пускать по SSH; для исправления этого достаточно один раз перезагрузить её).</p>
<p>Роль интерфейса между радиосетью и MQTT выполняет программа /usr/bin/mqtt, MQTT-брокер — mosquitto.</p>
<p>По умолчанию программа mqtt запущена в режиме демона, чтобы остановить её, необходимо выполнить команду</p>
<pre><code>/etc/init.d/mqtt.lora stop
</code></pre>

<p>Ручной запуск mqtt можно использовать, чтобы получить больше отладочной информации. При работе mqtt в режиме демона отладочная информация выводится в системный лог, который можно посмотреть командой logread (или logread -f для непрерывного просмотра).</p>
<p>В MQTT-брокере сообщения попадают в топик devices/lora, для подписки на него необходимо в отдельном окне запустить:</p>
<pre><code>mosquitto_sub -t &quot;devices/lora/#&quot; -q 1
</code></pre>

<p>Сообщения от конкретного устройства будут попадать в субтопики с названием, идентичным адресу EUI64 данного устройства.</p>
<p>Тело сообщения имеет формат JSON и состоит из двух частей — данных и статуса устройства:</p>
<pre><code>{
&quot;data&quot;:
{
&quot;joined&quot;: 1,
&quot;class&quot;: &quot;C&quot;
},
&quot;status&quot;:
{
&quot;devEUI&quot;: &quot;807B859020000177&quot;,
&quot;rssi&quot;: 0,
&quot;temperature&quot;: 0,
&quot;battery&quot;: 0,
&quot;date&quot;: &quot;2017-02-21T13:02:21.147555Z&quot;
}
}
</code></pre>

<p>Значение полей статуса:</p>
<ul>
<li><code>devEUI</code> — EUI64 устройства, от которого пришло сообщение, в 16-ричной форме</li>
<li><code>rssi</code> — сила сигнала, с которой базовая станция приняла сообщение модуля, в дБм</li>
<li><code>temperature</code> — температура чипа конечного устройства</li>
<li><code>date</code> — время приёма сообщения по часам БС, часовой пояс UTC</li>
<li><code>battery</code> — напряжение питания устройства, мВ</li>
</ul>
<p>В случае, если значение <code>battery</code> равно 0, значения <code>rssi</code> и <code>temperature</code> также не должны учитываться (и они также будут равны 0).</p>
<p>Для отправки сообщения на устройство необходимо опубликовать его в топике данного устройства (данное сообщение установит период опроса датчиков температуры на 2 минуты на устройстве с адресом 807B859020000001):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/807B859020000001/lmt01&quot; -m &quot;set_period 2&quot;
</code></pre>

<p>В некоторых случаях бывает удобно разнести приём и отправку сообщений в разные топики. В таком случае надо в файле /etc/lora-mqtt/mqtt.conf установить параметр:</p>
<pre><code>mqtt_sepio = true
</code></pre>

<p>При следующем запуске mqtt входящие и исходящие сообщения разделятся:</p>
<pre><code>devices/lora/807B85902000016D/mosi/sht21 get
devices/lora/807B85902000016D/miso/sht21 { &quot;data&quot;: { &quot;temp&quot;: 24.63, &quot;humid&quot;: 14 }, &quot;status&quot;: { “devEUI”: 807B85902000016D, &quot;rssi&quot; : -15, &quot;temperature&quot; : -10, &quot;battery&quot; : 3250, &quot;date&quot; : &quot;2017-02-21T13:02:21.147555Z&quot; }}
</code></pre>

<p>Сообщения от БС к конечному устройству попадут в топик <code>mosi</code>, от конечного устройства к БС — в топик <code>miso</code>.</p>
<h2 id="unwd-base-r2">Работа с базовой станцией UNWD-BASE R2<a class="headerlink" href="#unwd-base-r2" title="Permanent link">&para;</a></h2>
<p>Базовая станция второй версии собрана на встраиваемом микрокомпьютере архитектуры ARM и не имеет поддержки Wi-Fi. Для подключения к базовой станции доступны два порта Ethernet — LAN с DHCP-сервером с собственным адресом 192.168.97.1 и WAN с DHCP-клиентом.</p>
<p>Питание базовой станции осуществляется по интерфейсу PoE с напряжением 48 В.</p>
<h2 id="unwd-base-r1_1">Обновление прошивки базовой станции UNWD-BASE R1<a class="headerlink" href="#unwd-base-r1_1" title="Permanent link">&para;</a></h2>
<p>Для обновления прошивки базовой станции требуется:</p>
<ol>
<li>Подключиться к включённой базовой станции по Wi-Fi или Ethernet клиентом SFTP (например, WinSCP)</li>
<li>Загрузить новый файл прошивки <code>openwrt-ar71xx-generic-unwone-squashfs-sysupgrade.bin</code> в папку <code>/tmp</code></li>
<li>Из консоли базовой станции выполнить команду</li>
</ol>
<pre><code>sysupgrade -v /tmp/openwrt-ar71xx-generic-unwone-squashfs-sysupgrade.bin
</code></pre>

<ol>
<li>Если одновременно требуется сбросить все настройки базовой станции (кроме настроек её радиомодема), то к команде необходимо добавить ключ -n</li>
</ol>
<p>В случае проблем с доступом к консоли базовой станции (например, при указании неверной сети Wi-Fi и отсутствии подключения по Ethernet) возможны несколько варианта восстановления:</p>
<ol>
<li>Сброс настроек на заводские. Включить базовую станцию, дождаться окончания её загрузки (светодиод Ready перестаёт мигать и горит непрерывно), нажать и более 10 секунд удерживать кнопку Reset</li>
<li>Сброс настроек на заводские, вариант второй. На выключенной базовой станции нажать кнопку Reset и, удерживая её, подать питание. Дождаться, когда светодиод Ready мигнёт четвёртый раз, и отпустить кнопку Reset.</li>
<li>Обновление прошивки с флэшки с одновременным сбросом настроек.<ul>
<li>На отформатированной в FAT32 USB-флэшку создать папку _fw и положить в неё файл openwrt-ar71xx-generic-unwone-squashfs-sysupgrade.bin</li>
<li>Вставить флэшку в разъём USB базовой станции при выключенном питании</li>
<li>Нажав и удерживая кнопку Reset, подать питание</li>
<li>Дождаться, пока светодиод Ready ярко мигнёт один раз, после чего отпустить кнопку Reset</li>
<li>Дождаться обновления прошивки и перезагрузки базовой станции (около 1 минуты)</li>
</ul>
</li>
</ol>
<p>После обновления прошивки перед заходом на базовую станцию по SSH её необходимо один раз перезагрузить, выключив и снова включив питание.</p>
<h2 id="unwd-base-r2_1">Обновление прошивки базовой станции UNWD-BASE R2<a class="headerlink" href="#unwd-base-r2_1" title="Permanent link">&para;</a></h2>
<p>В базовой станции версии 2 используется встраиваемый компьютер на базе SoC iMX287 с увеличенным объёмом флэш-памяти и файловой системой UBI, что затрудняет процедуру обновления прошивки и делает её более рискованной. В связи с этим не рекомендуется без надобности обновлять прошивку базовой станции — в абсолютном большинстве случаев достаточно обновления одной лишь утилиты <a href="https://files.unwds.com/files/range/lora-mqtt/">lora-mqtt</a>.</p>
<p>Тем не менее, полное обновление прошивки возможно по следующей процедуре:</p>
<ol>
<li>
<p>Зайдя в консоль компьютера базовой станции, проверьте, доступна ли вам утилита nandwrite. Если она отсутствует — загрузите на БС и установите командой <code>opkg install</code> пакет <a href="https://files.unwds.com/files/range/imx287/busybox_1.23.2-1_mxs.ipk">busybox</a></p>
</li>
<li>
<p>Скачайте в каталог /tmp образ прошивки <a href="https://files.unwds.com/files/range/imx287/ubi.img">ubi.img</a></p>
</li>
<li>
<p>Выполните следующую последовательность команд:</p>
</li>
</ol>
<pre><code>cp /bin/busybox /tmp
/tmp/busybox flash_eraseall /dev/mtd6
/tmp/busybox nandwrite -p /dev/mtd6 /tmp/ubi.img
</code></pre>

<ol>
<li>
<p>Сразу после завершения работы утилиты nandwrite перезагрузите БС.</p>
</li>
<li>
<p>После перезагрузки скачайте и установите с помощью <code>opkg update</code> свежую версию утилиты <a href="https://files.unwds.com/files/range/lora-mqtt/">lora-mqtt</a> (сборка для платформы <code>mxs</code>).</p>
</li>
</ol>
<p>Внимание! Команды должны выполняться строго в указанном виде, файл прошивки и копия утилиты busybox должны находиться только в папке /tmp!</p>
<p>Пользовательские настройки, скрипты и программы при обновлении прошивки не сохраняются.</p>
<p>Если также требуется обновление ядра ОС (файл <a href="https://files.unwds.com/files/range/imx287/uImage">uImage</a>) и описания оборудования DTB (файл <a href="https://files.unwds.com/files/range/imx287/imx28-duckbill.dtb">imx28-duckbill.dtb</a>), его произвести несколько проще:</p>
<pre><code>flash_eraseall /dev/mtd3
nandwrite -p /dev/mtd3 /tmp/uImage
flash_eraseall /dev/mtd4
nandwrite -p /dev/mtd4 /tmp/imx28-duckbill.dtb
</code></pre>

<h2 id="unwd-base-r2_2">Обновление прошивки радиомодулей UNWD-BASE R2<a class="headerlink" href="#unwd-base-r2_2" title="Permanent link">&para;</a></h2>
<p>Для обновления прошивки радиомодулей необходимо загрузить и установить на базвую станцию пакет stm32flash:</p>
<ol>
<li>Скачать файл <a href="https://files.unwds.com/files/range/lora-mqtt/stm32flash_0.5-2_mxs.ipk">stm32flash</a></li>
<li>Загрузить его на базовую станцию в директорию /tmp</li>
<li>Выполнить команду opkg install /tmp/stm32flash*</li>
</ol>
<p>Перед обновлением прошивки необходимо остановить утилиту lora-mqtt: '/etc/init.d/mqtt.lora stop'</p>
<p>Перевод радиомодуля в режим обновления производится последовательностью команд для первого модуля:</p>
<pre><code>echo 1 &gt; /sys/class/gpio/gpio98/value
echo 0 &gt; /sys/class/gpio/gpio99/value
echo 1 &gt; /sys/class/gpio/gpio99/value
echo 0 &gt; /sys/class/gpio/gpio98/value
</code></pre>

<p>Для второго модуля:</p>
<pre><code>echo 1 &gt; /sys/class/gpio/gpio106/value
echo 0 &gt; /sys/class/gpio/gpio107/value
echo 1 &gt; /sys/class/gpio/gpio107/value
echo 0 &gt; /sys/class/gpio/gpio106/value
</code></pre>

<p>В случае, если команды запускаются скриптом, рекомендуется вставить между командами паузы не менее 0,1 сек.</p>
<p>Далее необходимо загрузить файл прошивки (например, lora-star-uni-gate.hex) в директорию /tmp и выполнить команду (для второго радиомодуля порт меняется на /dev/ttyAPP1)</p>
<pre><code>stm32flash -b 115200 -E -R -w /tmp/lora-star-uni-gate.hex /dev/ttyAPP0
</code></pre>

<p>Прошивка занимает около 15 секунд, после чего радиомодуль автоматически перезагрузится.</p>
<p>Внимание! Штатная версия утилиты stm32flash (0.4), входящая в состав OpenWRT 15.05.1, равно как и последняя на данный момент версия 0.5 с оригинального сайта разработчика некорректно работают с используемыми в радиомодулях Unwired Range микроконтроллерами.</p>
<p>Для прошивки необходимо использовать только модифицированную версию stm32flash. Если вам необходимо ознакомиться с её исходными кодами, они доступны на <a href="https://github.com/unwireddevices/stm32flash">GitHub</a>.</p>
<h2 id="lora-mqtt">Обновление ПО lora-mqtt базовой станции<a class="headerlink" href="#lora-mqtt" title="Permanent link">&para;</a></h2>
<p>Часто достаточно обновить только программу lora-mqtt, не затрагивая всю прошивку базовой станции.</p>
<p>Для этого необходимо с помощью scp (например, программы WinSCP) подключиться к базовой станции по сети и загрузить файл lora-mqtt-*.ipk в папку /tmp, после чего зайти
на базовую станцию по SSH и выполнить команды:</p>
<pre><code>/etc/init.d/mqtt.lora stop
opkg install /tmp/lora-mqtt*.ipk
</code></pre>

<p>Если после второй команды в консоли не появится сообщение «Attempting to run in the background» — то для запуска новой версии lora-mqtt в качестве постоянно работающего сервиса надо перезагрузить базовую станцию или выполнить команду</p>
<pre><code>/etc/init.d/mqtt.lora start
</code></pre>

<p>Для базовой станции версии R1 используются пакеты архитектуры ar71xx, для версии R2 — mxs.</p>
<h2 id="_4">Настройка радиомодулей базовой станции<a class="headerlink" href="#_4" title="Permanent link">&para;</a></h2>
<p>Начиная с прошивки радиомодуля базовой станции 1.25 и версии lora-mqtt 2.2.0, базовые станции UNWD-BASE R2.02 и новее поддерживают возможность удалённой настройки радиомодуля через API MQTT и специальный топик `devices/lora/gate.</p>
<p>Установка ключа шифрования сети (32 символа 16-ричной кодировки без лидирующего 0x):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/joinkey&quot; -m &quot;set AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA&quot;
</code></pre>

<p>Установка региона (всегда указывайте двузначное число; 00 — Европа, 01 — Россия):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/region&quot; -m &quot;set 01&quot;
</code></pre>

<p>Установка скорости связи (всегда указывайте двузначное число):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/datarate&quot; -m &quot;set 03&quot;
</code></pre>

<p>Установка номера канала (всегда указывайте двузначное число):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/channel&quot; -m &quot;channel 00&quot;
</code></pre>

<p>Перезагрузка радиомодуля (необходима после завершения настройки):</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/reboot&quot; -m &quot;&quot;
</code></pre>

<p>Перевод радиомодуля в режим обновления прошивки:</p>
<pre><code>mosquitto_pub -t &quot;devices/lora/gate/update&quot; -m &quot;&quot;
</code></pre>

<h2 id="raspberry-pi">Использование в качестве базовой станции Raspberry Pi<a class="headerlink" href="#raspberry-pi" title="Permanent link">&para;</a></h2>
<p>Для подключения модема к Raspberry Pi 3 можно использовать специальный адаптер, устанавливающийся в разъём RPi.
Первый вывод контакта адаптера должен совпадать с первым выводом разъёма RPi, модем в адаптер устанавливается чипами вверх.</p>
<p><img alt="Подключение Raspberry Pi" src="images/SAM_7451.jpg" /></p>
<p>На Raspberry Pi устанавливается образ Raspbian Jessie Lite выпуска марта 2017 или новее.</p>
<p>Далее доустанавливаются необходимые пакеты:</p>
<pre><code>sudo apt-get install mosquitto libmosquitto1 libmosquitto-dev -y
sudo apt-get install libc-ares-dev libssl-dev libxcrypt-dev -y
sudo apt-get install git -y
</code></pre>

<p>Скачивание и сборка программы lora-mqtt:</p>
<pre><code>git clone https://github.com/unwireddevices/lora-mqtt.git
cd lora-mqtt
make
sudo cp bin/mqtt /usr/bin
sudo chmod 755 /usr/bin/mqtt
sudo mkdir /etc/lora-mqtt
sudo cp dist/openwrt/files/mqtt.conf /etc/lora-mqtt/
</code></pre>

<p>В файле /etc/lora-mqtt/mqtt.conf необходимо изменить название порта с <code>/dev/ttyATH0</code> на <code>/dev/ttyS0</code>.</p>
<p><em>NB: если у вас Raspberry Pi 2 или ещё более старая модель, порт должен называться <code>/dev/ttyAMA0</code>.</em></p>
<p>Кроме того, либо после настройки Raspberry Pi, либо при создании образа Debian Jessie на карте microSD необходимо изменить два конфигурационных файла.</p>
<ol>
<li>В файл config.txt (/boot/config.txt в настроенной RPi) добавить в конце строку <code>enable_uart=1</code></li>
<li>В файле cmdline.txt (/boot/cmdline.txt в настроенной RPi) в строке <code>dwc_otg.lpm_enable=0 console=serial0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait</code> полностью удалить параметр <code>console=serial0,115200</code></li>
</ol>
<p>Если файлы изменялись на работающей RPi, после изменения плату необходимо перезагрузить. Эти изменения включат внешний порт UART, а также отключат вывод в него консоли операционной системы.</p>
<p>Запускается программа командой <code>sudo mqtt</code>.</p></div>
